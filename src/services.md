# 接案合作

<p class="service-lead">
  我目前開放的合作形式，偏向需求相對明確、可以在一個專案週期內完成交付的小型系統或內部工具。
</p>

我不是以長期駐點維護為主的接案模式。
如果你需要的是一個能實際解決流程問題的工具，並且希望交付後由你們內部自行使用或接手，那會比較適合和我合作。

<div class="service-note">
  我特別適合處理「原本靠 Excel、人工訊息、紙本或口頭協調」的流程，整理成一個可操作、可追蹤、可交付的小系統。
</div>

我比較熟悉的交付形式，是以 Web 系統為主的內部工具。這類型通常比較容易部署、操作，也比較適合中小型團隊落地使用。

## 我的工作方式

1. 先確認你要解決的是哪一段流程，而不是一開始就堆功能。
2. 把需求整理成可實作的畫面、資料與規則。
3. 以能交付、能上線、能被使用為目標，控制複雜度。
4. 專案結束時，盡量讓系統維持在你們可理解、可接手的狀態。

## 我比較不適合的案子

- 長期駐點維運。
- 規模太大、需求持續變動但沒有決策窗口的專案。
- 需要大量行銷頁面、品牌設計或高互動前端特效的網站。
- 期待用極低成本完成高複雜度產品的案型。

## 如果你正在找什麼樣的人

如果你希望找到的是：

- 能把需求講清楚、再開始做的人。
- 願意處理資料結構、規則整理與流程梳理的人。
- 比起炫技，更重視交付品質與實際可用性的人。

那這類合作通常會很順。

## 聯絡前可以先準備的資訊

1. 你目前是用什麼方式在處理這件事。
2. 最痛的地方是什麼，最想先解決哪一段。
3. 預期給多少人用，是否需要帳號、權限、報表或匯出。
4. 這個工具希望一次做完，還是先做第一版。

這些資訊越清楚，越容易快速判斷是否適合合作。
