## 我的切入角度

很多工程師是從資訊本科走進業界，我不是。

我原本就讀心理學相關科系，後來轉向軟體工程。這個背景改變了我看待系統的方式：
我會自然地從人的需求、行為脈絡與溝通成本出發，去思考一個功能為什麼存在、它該怎麼被使用，以及它會在哪裡讓人卡住。

這讓我在面對內部行政系統、報表流程或跨角色溝通時，不會只停在技術語言，而會試著把問題還原成：

1. 使用者真正想完成什麼。
2. 團隊目前卡住的是流程、資訊，還是系統設計。
3. 哪一種實作方式，最能兼顧正確性、維護性與交付速度。

## 我做過的事情

### 全端系統開發

- 以 ASP.NET、ASP.NET Core、Vue、jQuery 為主要開發工具。
- 處理過校務行政系統、報表系統、環保相關網站等產品型專案。
- 熟悉 SQL Server、PostgreSQL、Entity Framework、Dapper。
- 有 RWD 介面、既有系統維護、效能調整與問題排查經驗。

### 軟體工程方法的持續深化

- 長期關注 Clean Code、Clean Architecture、DDD、Design Pattern、BDD。
- 不把這些名詞當作口號，而是拿來改善模組邊界、需求落差與開發效率。
- 在團隊中參與新人培訓、技術分享與 Code Review。

### 把技術放回產品脈絡

- 我習慣先釐清需求，再選擇架構與實作方式。
- 面對流程複雜的系統時，我偏好從 Top-Down 角度規劃畫面、資料與規則。
- 我也樂於與 AI 協作，讓探索、重構與文件整理的速度更高，但仍維持工程判斷與品質標準。
